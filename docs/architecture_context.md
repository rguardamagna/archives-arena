# 🗺️ TubeRPG: Master Context & E2E Blueprint

## 1. Product Vision
An E2E web-based Gamified Learning Platform (RPG style) where educational YouTube videos act as "Levels". The user studies the video and faces dynamic multiple-choice "Battles" generated by AI based strictly on the video's transcript.

## 2. The User Journey (Frontend Dynamics)
1. **Lobby:** The user logs in and sees their Player Profile (Current HP, Level) and a list of available Playlists (e.g., Brais Moure's Python Course).
2. **The Level (Battle UI):** The user selects a video. The YouTube iframe loads on the left. On the right, the Enemy stats appear.
3. **The Engagement:** At specific intervals (or when the user hits "Challenge"), a popup interrupts the video with a Multiple Choice Question generated by the AI Brain.
4. **The Resolution:** 
   - *Correct Answer:* Attack animation! Enemy loses HP.
   - *Wrong Answer:* Damage animation! Player loses HP.
5. **Level Clear:** When Enemy hits 0 HP, the user gains XP/Level up and unlocks the next video.

## 3. High-Level Architecture (E2E)
- **Frontend (Presentation):** React + Vite. Handles the YouTube Player API, Battle Animations, and State Management.
- **Backend (API):** Python + FastAPI. Hexagonal Architecture (Domain, Application, Infrastructure).
- **External Ports:**
  - *Video Source:* `youtube-transcript-api` (YouTube Adapter).
  - *Brain:* Gemini 3.1 Pro (Generates structured MCQs via Native Adapter).
  - *Persistence:* Google Cloud Firestore (Saves HP, Player Profiles, Progress).
